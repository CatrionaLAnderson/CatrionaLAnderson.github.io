Aware Phone Interface - Ongoing project

Overview

Aware Phone Interface was my Final Year Project, completed in May 2025, designed to enhance user safety and experience when under the influence of alcohol. The system introduces a feature called "Drunk Mode"—similar in concept to "Do Not Disturb"—which activates when the user is intoxicated. It allows users to preconfigure safety preferences that guide their phone interactions while drunk. The interface aims to reduce cognitive load, prevent errors, and support safer phone use through a combination of real-time detection, UI changes, and health tools informed by UX research and user testing.

Key Features

  • Drunk Mode Activation: Users can toggle Drunk Mode manually, which locks in safety settings and adapts the phone UI.
  • Real-Time Typo Detection: An algorithm detects and flags potentially incoherent messages, suggesting corrections.
  • Accessible UI Layout: Simplified home screen with enlarged buttons and quick access to emergency features.
  • Health Support: BAC estimator and alcohol intake recommendations based on user BMI.
  • Custom Alerts: Users can configure custom notifications (e.g. “Don’t text your ex”) to trigger during Drunk Mode.

Technologies Used

  • Frontend: React Native, Expo
  • Backend: Node.js, Supabase
  • Language: JavaScript
  • Design: Figma
  • Methodology: Agile Development

Final Report

Final Reflections

Completing this year-long solo project was both challenging and rewarding. I developed the interface from the ground up—from UX research and prototyping to full-stack development and testing. One of the most valuable aspects was the user research phase, where survey feedback shaped key design decisions. This project deepened my interest in affective computing and UX design, and confirmed my passion for building technology that responds to real human needs.
